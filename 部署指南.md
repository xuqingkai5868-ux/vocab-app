### 走「本地 + tunnel」的好处
- 5 分钟跑起来，不需要登录腾讯云控制台
- 调试方便：改 `index.html` 刷新页面即生效
- cloudflared 不要求账号

### 走「正式部署到 EdgeOne Pages」的好处
- 数据持久化（KV Storage）
- 不依赖你电脑开机
- URL 是固定的（不会像 tunnel 那样每次重启变）

## ⚠️ 硬性前置（缺一个就部署不了）

### 前置 1：安装 edgeone CLI

在你的**本机**（不是沙箱）执行：

```bash
npm install -g edgeone
```

> macOS / Linux 必要时加 `sudo`。
> Windows 用 PowerShell 或 Git Bash 都行。

验证装好：

```bash
edgeone --version
```

应输出版本号（如 `1.x.x`）。

---

### 前置 2：登录 CLI

```bash
edgeone login
```

按提示在浏览器里完成腾讯云账号授权。完成后：

```bash
edgeone whoami
```

应看到你的腾讯云账号信息。

---

### 前置 3：EdgeOne 控制台开 KV Storage（**关键，不开等于白部署**）

KV 是后端存储所有账号 / 进度的地方。**没开 KV → 6 个 Edge Function 全部报错。**

#### 3.1 创建 KV 命名空间

1. 打开 https://console.cloud.tencent.com/edgeone/pages
2. 左侧菜单 → **KV 存储** → **创建命名空间**
3. 命名：`vocab-app-kv`（或任意名字，记下来）
4. 区域：**选「全球」**（不区分孩子在哪访问）
5. 点 **确定** → 看到命名空间 ID 即可（暂时不用复制）

#### 3.2 创建 Pages 项目

1. 同控制台 → **Pages 应用** → **创建应用**
2. 选 **「导入现有项目」** 或 **「从 Git 导入」**（推荐 Git）
   - 如果选 **「直接上传」**：把 `vocab-app/` 整个目录打成 zip 上传
   - 如果选 **「从 Git 导入」**：先把这目录推到一个空 Git 仓库
3. 项目名：`vocab-app`（或任意，最终 URL 会带这个名）
4. 构建命令：**留空**（纯静态 HTML，不需要构建）
5. 输出目录：**留空**（默认就是根目录）
6. 创建

#### 3.3 绑定 KV 到项目（**最易忘的一步**）

1. 进入项目详情页 → 左侧 **「环境变量」** 或 **「KV 绑定」**
2. 找到 **「KV 命名空间绑定」** → **添加绑定**
3. 填写：
   - **变量名（key）**：`my_kv` ← **必须填这个，Edge Function 里就是用 `my_kv` 这个全局名访问**
   - **命名空间（value）**：选你刚创建的 `vocab-app-kv`
4. 保存

> 💡 **如果以后想改 KV 名字**：要同时改所有 6 个 Edge Function 里的 `my_kv` → 你的新名字。改完重新部署。

## 🚀 部署流程

### 步骤 1：本地调试（可选但推荐）

先在本地跑起来试一下：

```bash
cd vocab-app
export PAGES_SOURCE=skills   # macOS/Linux
# Windows PowerShell: $env:PAGES_SOURCE="skills"
edgeone pages dev
```

正常会输出：

```
✓ Local server: http://localhost:8088
```

浏览器打开 `http://localhost:8088` 看到登录页 → 说明前端 + Edge Function 都跑起来了。

> ⚠️ **首次 dev 可能慢**（要下载 runtime 镜像）。耐心等 1-2 分钟。
> ⚠️ 本地 dev 时**不需要走鉴权**（middleware 在 dev 环境通常跳过），但 6 个 API 接口都可达。

---

### 步骤 2：正式部署

```bash
cd vocab-app
edgeone pages deploy
```

正常输出：

```
✓ Build complete
✓ Deploy complete
✓ URL: https://vocab-app-xxxx.edgeone.app
```

把那个 URL 记下来。

---

### 步骤 3：首次 Seed（**必须，否则 3 账号都不存在**）

部署完后，**3 个账号的 PIN 还没设**。需要手动调一次 `/api/seed`。

#### 3.1 准备 Seed 请求体

在你的电脑本地随便新建一个文件 `seed.json`：

```json
{
  "users": {
    "gao":   { "name": "哥哥", "role": "user", "grade": "高一" },
    "di":    { "name": "弟弟", "role": "user", "grade": "三年级" },
    "admin": { "name": "管理员", "role": "admin" }
  },
  "pins": {
    "gao":   "1111",
    "di":    "2222",
    "admin": "9999"
  }
}
```

> ⚠️ **生产环境请改 PIN！** 别用 1111/2222/9999。

#### 3.2 用 curl 调 seed

把下面的 `https://你的URL.edgeone.app` 换成步骤 2 拿到的 URL：

```bash
curl -X POST "https://你的URL.edgeone.app/api/seed" \
  -H "Content-Type: application/json" \
  -d @seed.json
```

正常返回：

```json
{
  "ok": true,
  "seeded": ["gao", "di", "admin"],
  "overwritten": false
}
```

> 💡 **如果想重置 3 账号**（比如换 PIN），重发同样请求，但加上 `"adminConfirmPin": "9999"`：
> ```bash
> curl -X POST "https://你的URL.edgeone.app/api/seed" \
>   -H "Content-Type: application/json" \
>   -d '{"users":{...},"pins":{...},"adminConfirmPin":"9999"}'
> ```

### 步骤 4：访问测试

浏览器打开 `https://你的URL.edgeone.app`，应该看到：

- 登录页：3 个身份按钮（哥哥 / 弟弟 / 管理员）
- 点 **哥哥** → 输 PIN `1111` → 进首页 → 看到 Day 1 新词
- 点 **管理员** → 输 PIN `9999` → 进首页 → 顶部多了「用户切换」按钮 + 底部「所有用户详情」

**3 账号互不可见规则**：
- 哥哥登录 → 只能看到自己的进度 + 弟弟的百分比
- 弟弟登录 → 只能看到自己的进度 + 哥哥的百分比
- 管理员登录 → 看到所有人详情，并能切换查看

## 🔧 日常维护

### 重置某个孩子的进度

孩子自己登录 → 底部「重置」按钮 → 确认。

如果你（管理员）想帮孩子重置 → 登录管理员 → 顶部「用户切换」点到孩子 → 底部「重置」。

### 导出 / 导入备份

- **导出**：底部「导出」按钮 → 下载 `vocab-backup-{userId}-{date}.json`
- **导入**：底部「导入」按钮 → 选 JSON 文件 → 确认覆盖
  - ⚠️ 导入文件必须与当前账号匹配（userId 字段要一致）

### 修改 PIN

只能通过**重新 seed**：

```bash
curl -X POST "https://你的URL.edgeone.app/api/seed" \
  -H "Content-Type: application/json" \
  -d '{
    "users": {
      "gao":   { "name": "哥哥", "role": "user", "grade": "高一" },
      "di":    { "name": "弟弟", "role": "user", "grade": "三年级" },
      "admin": { "name": "管理员", "role": "admin" }
    },
    "pins": {
      "gao":   "新PIN",
      "di":    "新PIN",
      "admin": "新PIN"
    },
    "adminConfirmPin": "老PIN"
  }'
```

> 不会清空进度（只更新 PIN + 账号资料）。想清空进度用 reset 按钮。

## ❓ 故障排除

### Q1: 部署后打开页面是 404

**A**: 大概率是 `index.html` 没在根目录。检查：
- `vocab-app/index.html` ✓
- `vocab-app/dist/index.html` ✗（构建后位置不对）

如果是后者，在 Pages 项目设置里把「输出目录」改成 `dist`。

---

### Q2: 登录时一直报 "network_error"

**A**: Edge Function 没跑起来。最常见原因：**忘了绑 KV**（前置 3.3）。去控制台检查。

---

### Q3: 登录时报 401 / "login_failed"

**A**: 没 seed 或 PIN 输错。重发一次 seed 请求（带 `adminConfirmPin`），再试。

---

### Q4: 管理员看不到「用户切换」按钮

**A**: 登录的账号 role 不是 admin。检查 seed 请求里 `admin` 账号的 `role: "admin"`。重新 seed。

---

### Q5: edgeone 命令找不到

**A**: CLI 没装好或 PATH 没生效。重装：
```bash
npm install -g edgeone
# 然后新开一个终端窗口（PATH 才生效）
```

---

### Q6: KV 绑定后还是报错 "my_kv is not defined"

**A**: 变量名打错了。**必须是 `my_kv`**，一字不差。改完要**重新部署**（不是只保存环境变量，要再跑 `edgeone pages deploy`）。

---

### Q7: 想完全清空重新开始

1. 后台：删 KV 命名空间 → 重建同名 KV
2. 重新部署
3. 重发 seed 请求
